# Product Thinking: UX critique to A/B Test

A practice project for building product thinking and UI/UX critique skills — starting from
analyzing an existing product's UX flow, identifying friction points, and progressing to
designing and running an A/B test to validate a proposed fix.

## Overview

The project moves through the full loop of applied product thinking:
- **Critique** — pick a real flow, break down the UX (information architecture, friction, visual design)
- **Hypothesize** — frame a specific problem and a proposed change, grounded in user needs
- **Test** — design an A/B test: metric, variants, sample size, success criteria
- **Learn** — document outcomes and what they imply for the next iteration
